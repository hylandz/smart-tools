# JT808协议解析工具
## 版本号结构

`V1.0.0` 由三部分数字组成，用点号分隔：
**`主版本号.次版本号.修订号`**
（即 `MAJOR.MINOR.PATCH`）

1. 主版本号（MAJOR）：
   - 表示产品或软件有**重大功能更新**或**不兼容的API变更**。
   - 从 `0.x.x` 到 `1.0.0` 通常意味着产品从开发阶段（Alpha/Beta）进入稳定阶段，适合正式发布。
2. 次版本号（MINOR）：
   - 表示**向后兼容的功能新增**或改进。
   - 在 `1.0.0` 中为 `0`，说明这是首个稳定版本，尚未添加次要功能。
3. 修订号（PATCH）
   - 表示**向后兼容的Bug修复**或小改进。
   - 在 `1.0.0` 中为 `0`，说明发布时未发现需要紧急修复的问题。
